# 个人作品集/简历主页

## 简介

展示个人项目、技能、联系表单的个人作品集网站。包含动画效果和响应式布局，支持一键部署到 GitHub Pages 或 Netlify。作为全栈学习路线的第一个项目，目标是快速产出一个可展示的完整静态站点。

## 技术栈

- 前端：HTML5、CSS3、JavaScript（纯静态）
- 后端：无（纯前端）
- 数据库：无
- 中间件：无
- 对象存储：无（图片内联 Base64 或放 assets/）
- 部署：GitHub Pages
  
  ## 目录结构
  
- portfolio-website
  ├── index.html # 主页面（个人介绍 + 技能 + 项目卡片 + 联系表单）
  ├── css/
  │ └── style.css # 全局样式、响应式布局、动画
  ├── js/
  │ └── main.js # 交互逻辑、表单校验、滚动动画
  ├── assets/
  │ └── images/ # 头像、项目截图等静态资源
  └── README.md
  
  ## 功能特性
  
- 编写中：个人简介与技能标签展示
- 编写中：项目卡片展示区，含截图与链接
- 编写中：联系表单（前端校验 + formspree/Netlify Forms 对接）
- 编写中：响应式布局（移动端 / 平板 / 桌面）
- 编写中：CSS 动画（滚动渐入、hover 效果）
- 计划中：国际化（中/英双语切换）
- 计划中：小程序版名片，展示技能与项目
  
  ## 快速开始
  
  ### 环境要求
  
- 任意现代浏览器
- （可选）Node.js 用于本地开发服务器（Live Server）
  
  ### 本地运行
  
  克隆仓库后直接用浏览器打开 index.html
  或使用 VS Code Live Server 插件启动本地服务器
  推送 main 分支到 GitHub 后，在 Settings > Pages 中启用部署
  
  ## 环境变量说明
  
- 无（纯静态项目，无需环境变量）
  
  ## 测试
  
- 手动测试：在不同浏览器及移动端设备上验证响应式布局
- （可选）Lighthouse 性能与可访问性审计
  
  ## 部署
  
- GitHub Pages：推送 main 分支，Settings > Pages 选择 main 分支，自动部署
- Netlify：连接 GitHub 仓库，自动构建部署
